Finding the new versions
------------------------
1. Check https://www.desmos.com/api/changelog?lang=en
   - The topmost entry is the latest stable release (e.g. v1.11.4)
2. Bump the minor version by 0.01 in the URL to find the prerelease (e.g. v1.11 → v1.12)
   - If that page exists, it is the current prerelease

Updating the files
------------------
3. Download both versions of the Desmos API source into lib/
4. Delete the old .js versions from lib/

5. In src/extension.js, update the two version constants near the top of openDesmosLocal():
   - `file`: the filenames of the downloaded scripts (prerelease and stable)
   - `versionLabel`: short display strings used in panel titles (e.g. 'v1.12-pre' / 'v1.11.4')

6. In src/extension.js, update the label strings inside DesmosDataProvider.getChildren():
   - The prerelease/stable toggle label and tooltip strings (search for 'v1.12' / 'v1.11.4')

7. In package.json, bump `"version"` (e.g. 0.5.0 → 0.5.1)

Testing & publishing
--------------------
8. Run: npm test   (all tests should pass)
9. Press F5 and select "VS Code Extension Development" to smoke-test in a live host

* For maintainer:
10. Run: npm install -g vsce
11. Run: vsce package
12. Run: vsce publish*** just upload the .vsix to the azure site, keys are too much of a pain in the ass to renew and work with
13. Stage, commit, & push
